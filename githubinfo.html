All About Github

  • Q. What is a version control system?
  • A. Tracks the history of changes as people work on projects together.
  • Q. What is Git?
  • A. An open source distributed version control system.
  • Q. What is GitHub?
  • A. A platform/cloud-based service for software development/version control using Git.
  • Q. What is the difference between Git and GitHub?
  • A. Git allows you to manage your source code history. Github is a hosting service so you can manage Git repositories.
  • Q. Who started GitHub and how was it started?
  • A. Chris Wanstrath, P.J. Hyett, Tom Preston-Werner, and Scott Chacon. It was a start-up business and they used Ruby on Rails.
  • Q. What company owns it now?
  • A. Microsoft.
  • Q. How much does a GitHub account cost?
  • A. It's free but you can do a Team subscription for $4 monthly or Enterprise for $21 monthly.
  • Q. What is the Octocat?
  • A. The mascot for Github.

Git/GitHub terms:

  1. repository: Contains project files and their revision history.
  2. commit: It's essentially the same as saving, but it records changes to the files.
  3. fork: A new repository that shares code and visibility settings with the original upstream repository.
  4. push: Uploads content from a local repository to a remote repository.
  5. pull requests: Tells others about changes that have been pushed to a branch in a repository.
  6. workflows: A configurable automated process that can run one or more jobs.
  7. issues: Allows you to track your work on GitHub.
  8. raw button: Lets you view/copy the raw content of a file without any styling.
  9. blame button: Used to examine contents of a file line by line and see when each one was modified and who changed it.
