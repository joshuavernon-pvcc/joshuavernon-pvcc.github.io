Chapter 6: Programming Languages

Term Definition
machine language The lowest-level programming language consisting of binary code that the computer's central processing unit (CPU) can directly execute.
debugging The process of identifying and fixing errors or bugs in a computer program.
assembly language A low-level programming language that uses symbolic names for machine-level instructions. It's specific to a particular computer architecture.
translator A program that translates high-level programming languages into machine code or assembly language.
compiler A translator that converts the entire high-level program into machine code before executing it. The resulting machine code is saved as an executable file.
interpreter A translator that converts high-level programming languages into machine code line by line during runtime. No separate executable file is generated.
cross-platform software Software that can run on multiple operating systems or computer architectures without modification.
natural language A human language used for communication, such as English or Spanish.
formal language A language with precise rules and structures, often used in programming and mathematics.
machine independence The ability of software to run on different types of hardware without modification.
high-level programming language A programming language with strong abstraction from the details of the computer's architecture.
object-oriented programming (OOP) A programming paradigm that uses objects, classes, and inheritance to structure and organize code.
scripting language A programming language designed for automating the execution of tasks, often in the context of software applications.
data types The classification or categorization of data into different types, such as integers, floats, and strings.
integer A data type representing whole numbers without fractional parts.
float A data type representing numbers with decimal points or fractions.
string A data type representing sequences of characters, typically used for text.
primitive data type A basic data type that is not composed of other data types.
data structure An organized way to store and manage data, such as arrays, linked lists, or trees.
Python collection data structures Specialized data structures provided by the Python programming language, including lists, tuples, sets, and dictionaries.
list A collection of ordered and mutable elements in Python.
tuple An ordered and immutable collection of elements in Python.
set An unordered collection of unique elements in Python.
dictionary A collection of key-value pairs in Python, providing efficient data retrieval.
function A reusable block of code designed to perform a specific task.
local variable A variable declared within a specific function or block of code, with limited scope.
global variable A variable declared at the top level of a program and accessible throughout the entire code.
source program The original program written by a programmer, typically in a high-level programming language.
object program The machine code or executable file generated by a compiler or interpreter from the source program.
